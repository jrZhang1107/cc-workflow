# /withScout

此命令通过将复杂任务分解为调查阶段和执行阶段来处理任务。它使用 `investigator` 代理在决定行动计划之前收集信息。

## 何时使用

- **使用场景:** 用户有一个复杂的请求，在进行更改之前需要理解代码库。
- **建议场景:** 用户的请求如果不先从多个文件或代码库的多个部分收集信息就无法完成。
- **示例:** "用户：添加 JWT 令牌刷新功能。"
- **示例:** "用户：弄清楚我们项目的认证逻辑，然后添加一个新的端点。"

## 动作 (Actions)

此命令遵循 **调查 -> 综合 -> 迭代/执行** 的工作流。

1.  **步骤 1: 拆解与计划**

    - 将用户的主要目标分解为一组清晰的、可独立调查的问题。
    - 将每组问题分配给不同的 `investigator` 代理（例如，前端调查员、后端调查员）。

2.  **步骤 2: 并行调查**

    - 使用 `Task` 工具并发启动多个 `investigator` 代理。
    - 每个调查员将研究其分配的问题并直接返回 Markdown 报告。

3.  **步骤 3: 综合与评估**

    - 合并所有调查员的报告，形成对系统的整体视图。
    - 识别信息中的关键连接、知识空白或冲突。

4.  **步骤 4: 迭代或执行**

    - **如果信息不足 (迭代):** 构思新一轮更具体的调研问题，并返回步骤 2。
    - **如果信息充足 (执行):** 进入行动阶段 (步骤 5)。

5.  **步骤 5: 行动阶段**

    - 基于收集到的全面信息，制定计划并使用 `worker` 代理执行用户的最终请求（例如，实现功能、修复 Bug）。

6.  **步骤 6: 总结与报告**
    - 在交付最终结果时，解释调查过程、关键发现以及为实现结果所采取的操作。
