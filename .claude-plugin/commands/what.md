# /what

此命令在内部使用，当用户的请求过于模糊而无法执行时触发。它会读取项目文档以了解背景，然后向用户提出有针对性的、基于选项的问题，以澄清其意图。

## 何时使用

- **使用场景:** 此命令通常由主助手 AI 使用，而不是由用户直接使用。当用户的提示含糊不清时（例如，“修复它”、“添加一个东西”）触发。
- **目标:** 将模糊的请求转化为具体、可操作的计划。

## 动作 (Actions)

1.  **步骤 1: 收集上下文**

    - 阅读位于 `<projectRootPath>/llmdoc/index.md` 的文档索引和其他高层级文档，以了解项目的目的、架构和功能。

2.  **步骤 2: 构思澄清问题**

    - 基于文档和用户的模糊请求，构思一组澄清问题。
    - 问题应尽可能基于选项，以引导用户达成特定结果。例如，与其问“你想做什么？”，不如问“你是想：(a) 添加新的 API 端点，(b) 修改现有功能，还是 (c) 修复 Bug？”。

3.  **步骤 3: 询问用户**

    - 使用 `AskUserQuestion` 工具向用户呈现问题。

4.  **步骤 4: 构思调查任务**
    - 根据用户的澄清回复，您的目标是构思一组具体的**调查问题**。
    - **不要急于得出解决方案。** 此命令的目的是澄清“用户想知道什么”，而不是“如何实现它”。
    - 调用 `/withScout` 命令并附上您构思的清晰、事实性的问题。例如，如果用户现在想要“添加用户端点”，下一步是让 `/withScout` 调查“当前的 API 路由结构是什么？”以及“定义数据模型使用什么规范？”。
