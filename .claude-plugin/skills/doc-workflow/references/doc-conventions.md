# 文档规范

## 代码引用格式

始终引用代码而不是粘贴：

```
`path/to/file.ext` (SymbolName): 简短描述
```

**示例：**
```
`src/auth/jwt.js` (generateToken, verifyToken): 处理 JWT 的创建和验证
```

## 文档质量检查清单

- [ ] **简洁性**: 是否在 150 行以内？
- [ ] **清晰性**: 从标题和前几行能看清目的吗？
- [ ] **准确性**: 是否基于经过验证的代码？
- [ ] **分类**: 是否在正确的类别中？
- [ ] **无代码块**: 是否仅使用了引用？

## 命名规范

| 类型 | 规范 | 示例 |
|------|-----------|---------|
| 文件 | kebab-case (短横线隔开) | `project-overview.md` |
| 类别 | 小写 | `overview/`, `guides/` |
| 标题 | 标题格式 (Title Case) | `# 项目概览` |

## 更新原则

1. **极简性**: 使用尽可能少的文字
2. **准确性**: 基于实际代码，而非假设
3. **一致性**: 与现有文档风格匹配
4. **原子性**: 每个文档仅包含一个概念
