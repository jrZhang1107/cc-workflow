# 调查指南

## 代码引用格式

引用代码时，始终使用此格式：

```
`path/to/file.ext` (FunctionName, ClassName): 简短描述
```

**正确示例：**
```
`src/auth/jwt.js` (generateToken, verifyToken): 处理 JWT 的创建和验证
```

**错误示例：**
```javascript
// 不要像这样粘贴代码块
function generateToken(payload) {
  // ... 50 行代码
}
```

## 调查深度指南

| 问题类型 | 方法 |
|--------------|----------|
| "什么是 X？" | 先阅读概览文档 |
| "X 是如何工作的？" | 先阅读架构文档，然后阅读代码 |
| "X 是在哪里定义的？" | 使用 Grep/Glob 定位 |
| "为什么 X 被设计成这样？" | 检查 git 历史和文档 |

## 报告结构

1. **代码部分**: 列出所有相关的代码位置
2. **结论**: 关键的事实性结论
3. **关系**: 组件之间如何连接
4. **结果**: 对问题的直接回答
