多 CLI 协作框架
for Claude Code

让 Gemini、Codex 等多个 AI 模型在 Claude Code 中并行协作， 配合 llmdoc 文档驱动开发，大幅提升复杂任务的执行效率。

快速安装 查看源码
$ npx cc-workflow

核心功能

专为 Claude Code 设计的多模型协作与文档驱动开发体系

🤝

多 CLI 并行协作

同时调度 Gemini、Codex 等多个 AI 模型，对同一任务进行并行分析或实现，结果自动聚合。

📚

llmdoc 文档驱动

自动生成并维护针对 LLM 优化的项目文档，让 AI 始终掌握最新的代码库上下文。

🤖

专业化 Agent 体系

内置 worker、investigator、recorder、scout 等多个专业 Agent，各司其职，精准执行。

⚡

Team 工作流

通过 team-research → team-plan → team-exec → team-review 四阶段流程，系统化完成复杂任务。

🔌

Claude Code 深度集成

作为 Claude Code 插件运行，提供斜杠命令和自动触发技能，无缝融入现有工作流。

📦

零外部依赖

纯 Node.js 实现，无任何第三方依赖，安装轻量，Node >= 18 即可运行。

安装方式

三种方式，按需选择

1

运行安装命令

# 一键安装到 Claude Code 插件目录
npx cc-workflow
2

在 Claude Code 中执行以下命令注册插件

/plugin marketplace add C:\Users\<你的用户名>\.claude\plugins\cc-workflow
/plugin install llmdoc-ccw@cc-workflow-marketplace
3

重启 Claude Code 即可使用所有命令和技能。

1

克隆仓库

git clone https://github.com/your-org/cc-workflow.git
cd cc-workflow
2

以插件目录模式启动 Claude Code

claude --plugin-dir ./llmdoc-ccw
1

在 Claude Code 中添加 GitHub marketplace 源

/plugin marketplace add your-username/cc-workflow
/plugin install llmdoc-ccw@cc-workflow-marketplace
2

重启 Claude Code 完成安装。

快速开始

安装后，几条命令即可体验完整工作流

# 1. 初始化 llmdoc 文档系统（首次使用）
/llmdoc-ccw:initDoc

# 2. 对复杂任务进行多 CLI 并行研究
/llmdoc-ccw:team-research 搭建用户认证模块

# 3. 生成分层执行计划
/llmdoc-ccw:team-plan

# 4. 并行执行所有任务
/llmdoc-ccw:team-exec

# 5. 双模型交叉验证结果
/llmdoc-ccw:team-review

# 或者：直接使用单 CLI 工具分析
/llmdoc-ccw:cli 分析认证模块的安全风险 --tool gemini

命令参考

完整的命令、技能与 Agent 列表

Commands（显式调用）

命令 描述 类型
/llmdoc-ccw:initDoc 初始化 llmdoc 文档系统，对项目进行深度调查并生成结构化文档 命令
/llmdoc-ccw:withScout 复杂任务专用：先深度调研代码库，再制定并执行计划 命令
/llmdoc-ccw:what 通过结构化问题澄清模糊请求，避免方向偏差 命令
/llmdoc-ccw:cli 调用 Gemini / Codex / Qwen 等多 CLI 工具进行分析或实现 命令
/llmdoc-ccw:team-research 多 CLI 并行分析代码库，生成约束集和依赖关系 命令
/llmdoc-ccw:team-plan 基于研究结果生成分层任务计划，构建依赖图 命令
/llmdoc-ccw:team-exec 按计划分层并行执行所有任务 命令
/llmdoc-ccw:team-review 双模型交叉验证执行结果，确保质量 命令

Skills（自动触发）

Skill 触发词 描述
investigate "什么是"、"分析"、"explain" 快速无状态代码库调查，直接在对话中返回结果
commit "提交"、"commit" 自动生成规范的 Git 提交信息
update-doc "更新文档"、"同步文档" 代码变更后自动更新 llmdoc 文档
read-doc "了解项目"、"读文档" 读取 llmdoc 概览，快速掌握项目上下文
team-workflow "team"、"并行执行"、"多CLI" 触发完整的 team 工作流协作模式
doc-workflow "文档工作流"、"llmdoc" 提供文档系统使用指导

Agents

Agent 用途
worker 精确执行明确定义的计划，严格遵守文件范围约束
investigator 快速无状态代码库分析，采用文档优先方法
recorder 创建并维护 llmdoc 文档，使用 4 类别结构（概览/指南/架构/参考）
scout 为 initDoc 命令执行深度调查，保存持久化报告
team-lead 协调多 Agent 协作，管理任务分配和执行流程
team-builder 执行具体的分析或实现任务，严格遵守文件范围约束
cli-execution-agent 使用多 CLI 工具（Gemini/Qwen/Codex）进行代码分析或实现
